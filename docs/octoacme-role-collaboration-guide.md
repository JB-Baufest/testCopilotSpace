# OctoAcme — Role Collaboration & Accountability Guide

## Purpose
Provide a lightweight reference that clarifies cross-role accountability, collaboration touchpoints, and handoff expectations during delivery.

## Lightweight responsibility matrix (RACI-style)
Legend: **A** = Accountable, **R** = Responsible, **C** = Consulted, **I** = Informed

| Activity | Product Manager | Project Manager | Scrum Master | UX Designer | Data Analyst | Developers | Stakeholder Proxy |
| --- | --- | --- | --- | --- | --- | --- | --- |
| Define problem statement & desired outcomes | A/R | C | I | C | C | I | C |
| Prioritize backlog and acceptance criteria | A/R | C | C | C | C | C | C |
| Plan sprint/iteration scope | C | A/R | R | C | I | R | I |
| Facilitate team ceremonies & remove blockers | I | C | A/R | I | I | C | I |
| Validate UX flows before implementation | C | I | I | A/R | I | C | C |
| Build, test, and review implementation | I | I | I | C | I | A/R | I |
| Track delivery/impact metrics and report insights | A | C | I | I | A/R | I | C |
| Consolidate stakeholder feedback and decisions | C | C | I | C | C | I | A/R |

## Cross-role collaboration cadence
- **Weekly delivery sync**: Project Manager, Product Manager, Scrum Master, tech lead/developer representative.
- **Design/product refinement**: Product Manager + UX Designer + Developers (as needed).
- **Metrics and outcome review**: Product Manager + Data Analyst + Project Manager.
- **Stakeholder alignment touchpoint**: Product Manager + Project Manager + Stakeholder Proxy.

## Handoff and review checklist
Use this checklist when work moves between roles or phases:

- [ ] Problem statement and expected outcome are documented.
- [ ] Scope and priority are confirmed by Product Manager.
- [ ] Delivery plan, dependencies, and risks are confirmed by Project Manager.
- [ ] UX assumptions/flows are validated for impacted user journeys.
- [ ] Metrics/KPIs for success are defined and owner is assigned.
- [ ] Blockers and escalation path are visible (owner + due date).
- [ ] Stakeholder Proxy has consolidated feedback and open decisions.
- [ ] Definition of Done and acceptance criteria are clear to Developers and QA/Testing.
